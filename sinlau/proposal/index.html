1. 專案背景與理解

打破資訊壁壘，實現全人照護

我們深刻理解新樓醫院「深耕台灣計畫」的宏觀願景。本專案的核心挑戰在於打破醫院、診所與病患之間的資訊壁壘，以應對日益增長的社區醫療及慢性病照護需求。

現有的 Portal 概念 (sinlau/) 提供了一個良好的起點。本建議書旨在將此概念擴展為一個功能完整、安全穩定且具高度擴展性的全人照護平台。

2. 專案四大核心目標

建構無縫的醫療照護生態系

資訊共享便捷化

醫院與診所間的病歷、檢查摘要與處置建議無縫流通。

政策資訊即時通

衛福部、健保署等最新政策與教育訓練即時推播。

個案管理智慧化

導入雙向通報，並透過 LINE 自動化提醒與衛教追蹤。

品質管理數據化

建立視覺化儀表板，提供異常提醒與品質比較分析。

3. 系統架構總覽

安全、整合的資訊交換樞紐

[醫院端]

透過安全的 API 閘道，上傳病歷摘要、檢查結果至雲端資料庫。

[中心平台 (本案核心)]

負責所有模組運算、數據分析、權限控管與資訊推播。

[診所端]

(Portal 介面) 登入平台查詢個案資訊、接收推播、進行線上照會。

[病患端]

透過 LINE 官方帳號，接收回診提醒、衛教資訊與健康處方。
