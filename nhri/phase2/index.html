可適性主動式關懷AI引擎

使用自然語言定義關懷規則。系統後端的 LLM 會將規則解析為可執行的觸發條件，實現自動化、個人化的主動關懷。

IF (規則觸發條件):
"當一位超過70歲、有高血壓病史的獨居使用者，連續三天沒有記錄血壓..."
THEN (自動執行動作):
"自動發送 Line 推播訊息: [陳伯伯，我先提醒您量一下血壓。記錄下來我幫您存檔。]"
IF (規則觸發條件):
"當使用者詢問 [睡不著] 或 [失眠] 關鍵字超過 3 次..."
THEN (自動執行動作):
"主動推播: [我找到一個針對長者的睡前舒眠音樂，現在播放給您聽好嗎？]"
BOT 設置服務 (任務型流程)

定義特定任務的對話流程，例如交通查詢、掛號協助等。

流程名稱: 交通查詢
觸發意圖: 詢問 (交通, "怎麼去", "車票")
流程:
  1. AI: "您想去哪裡？" (反問確認)
  2. User: "[目的地]"
  3. AI: "您想從哪裡出發？" (反問確認)
  4. User: "[出發地]"
  5. AI: (串接台鐵/高鐵/客運 API) -> "為您查詢到... (RAG 檢索個人化建議)"
